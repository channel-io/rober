# Rober Core Flow

## One-liner

> Rober는 티켓 시스템의 업무를 받아, 로컬 PC에서 사용자 승인 하에 자율적으로 처리하는 HITL 자동화 에이전트다.

## 핵심 구조

```
┌─────────────────────────────────────────────────────────────────┐
│                        EXTERNAL                                 │
│                                                                 │
│   ┌──────────────┐                                              │
│   │   Linear     │                                              │
│   │  (티켓 시스템) │                                              │
│   └──────┬───────┘                                              │
│          │                                                      │
│          │  티켓 발행 / 업무 요청                                   │
│          │  (단방향: push)                                        │
│          ▼                                                      │
└──────────┼──────────────────────────────────────────────────────┘
           │
           │  WebSocket / Port Forwarding
           │
┌──────────┼──────────────────────────────────────────────────────┐
│          ▼              LOCAL PC (사용자 업무 환경)                │
│                                                                 │
│   ┌──────────────────────────────────────────────────────┐      │
│   │                    ROBER                              │      │
│   │                                                      │      │
│   │   ┌────────────┐   ┌────────────┐   ┌────────────┐  │      │
│   │   │  Ingest    │──▶│  Planner   │──▶│  Executor  │  │      │
│   │   │ (티켓 수신) │   │ (작업 계획) │   │ (작업 실행) │  │      │
│   │   └────────────┘   └─────┬──────┘   └─────┬──────┘  │      │
│   │                          │                 │         │      │
│   │                          │    ┌────────┐   │         │      │
│   │                          └───▶│  HITL  │◀──┘         │      │
│   │                               │  Gate  │             │      │
│   │                               └───┬────┘             │      │
│   │                                   │                  │      │
│   └───────────────────────────────────┼──────────────────┘      │
│                                       │                         │
│                                 ┌─────┴─────┐                   │
│                                 │           │                   │
│                                 ▼           ▲                   │
│                            ┌────────────────────┐               │
│                            │      사용자         │               │
│                            │   (승인 / 방향 제시)  │               │
│                            └────────────────────┘               │
│                                                                 │
│   ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─   │
│                                                                 │
│   💤 PC 잠금 → Rober 대기 (티켓은 계속 쌓임)                       │
│   🔓 PC 잠금 해제 → 쌓인 티켓 정리 → 사용자에게 계획 보고            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 통신 방향

```
Linear ──────▶ Rober        단방향 (티켓 push)
Rober  ◀─────▶ 사용자       양방향 (승인, 질문, 방향 조정)
```

## 보안 모델

- 모든 판단과 실행이 **로컬 PC**에서 발생
- 외부 액션은 반드시 **사용자 승인(HITL Gate)** 통과
- PC 잠금 시 자동 대기 → 무인 실행 없음

## 핵심 사이클

```
[티켓 도착] → [Rober 인지] → [작업 계획 수립] → [사용자 승인 요청]
                                                      │
                                         ┌────────────┼────────────┐
                                         ▼            ▼            ▼
                                       승인          수정요청      거부
                                         │            │            │
                                         ▼            ▼            ▼
                                       실행        재계획         보류
                                         │            │
                                         ▼            │
                                      결과 보고 ◀──────┘
```
